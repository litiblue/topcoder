SRM 662 2 500 ExactTreeEasy

Problem Statement

You are given ints n and m. Construct any tree with exactly n nodes (labeled 0 through n-1) and exactly m leaves. For the given constraints on n and m a solution will always exist.

The return value should be a int[] containing a sequence of 2*(n-1) integers. If your tree has edges (a[0], b[0]), (a[1], b[1]), etc., the correct return value is the sequence {a[0], b[0], a[1], b[1], ...}. If there are multiple correct return values, you may return any of them.

Definition

  • Class ExactTreeEasy
  • Method getTree
  • Parameters integer , integer
  • Returns tuple(integer)
  • Method signature def getTree(self, n, m)
(be sure your method is public)

Limits

  • Time limit (s) 2.000
  • Memory limit (MB) 256

Notes

  • A tree is a connected acyclic graph. The leaf of a tree is a node of degree 1, i.e., a node that has exactly one outgoing edge.

Constraints

  • n will be between 3 and 50, inclusive.
  • m will be between 2 and n-1, inclusive.

Test cases

  1. input
    • n 4
    • m 2
    output
    Returns { 0, 1, 1, 2, 2, 3 }
    note
    We are supposed to construct a tree with 4 nodes and 2 leaves. One such tree is a simple path consisting of the edges 0-1, 1-2, and 2-3.
  2. input
    • n 4
    • m 3
    output
    Returns { 0, 1, 1, 2, 1, 3 }
    note
    Now we have to construct a tree with 4 nodes and 3 leaves. The tree returned by our solution contains the edges 1-0, 1-2, and 1-3. The leaves are the nodes 0, 2, and 3.
  3. input
    • n 3
    • m 2
    output
    Returns { 0, 1, 1, 2 }
  4. input
    • n 5
    • m 3
    output
    Returns { 0, 1, 1, 2, 1, 3, 3, 4 }
  5. input
    • n 10
    • m 9
    output
    Returns { 0, 1, 1, 2, 1, 3, 1, 4, 1, 5, 1, 6, 1, 7, 1, 8, 1, 9 }
